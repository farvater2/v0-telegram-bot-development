# Telegram Page Watcher Bot

Telegram-бот для автоматического мониторинга веб-страниц с поиском данных по регулярным выражениям и гибкими условиями оповещений.

## Возможности

- Мониторинг веб-страниц по расписанию
- Извлечение данных с помощью регулярных выражений
- Гибкие условия отправки уведомлений
- Поддержка двух режимов: проверка (check) и извлечение (extract)
- История проверок
- Экспорт задач в JSON

## Установка

### Требования

- Node.js 18+
- pnpm (или npm/yarn)

### Шаги установки

1. Клонируйте репозиторий:
```bash
git clone <repository-url>
cd telegram-page-watcher
```

2. Установите зависимости:
```bash
pnpm install
```

3. Создайте файл `.env` на основе `.env.example`:
```bash
cp .env.example .env
```

4. Получите токен бота у [@BotFather](https://t.me/BotFather) и добавьте в `.env`:
```
BOT_TOKEN=your_bot_token_here
```

5. Запустите бота:
```bash
# Режим разработки (с hot-reload)
pnpm dev

# Продакшн
pnpm build
pnpm start
```

| `/start` | Приветствие и инструкция |

| `/cancel` | Отмена текущей операции |

## Режимы работы

### Режим проверки (check)
Проверяет наличие данных на странице. Возвращает True/False.

### Режим извлечения (extract)
Извлекает конкретные значения по регулярному выражению.

## Условия отправки уведомлений

| Условие | Описание |
|---------|----------|
| `always` | При каждой проверке |
| `on_match` | Только при наличии совпадений |
| `on_change` | При изменении значения |
| `on_increase` | При увеличении числового значения |
| `on_decrease` | При уменьшении числового значения |

## Переменные в шаблонах

| Переменная | Описание |
|------------|----------|
| `{url}` | Адрес страницы |
| `{check_time}` | Время проверки |
| `{task_id}` | ID задачи |
| `{task_name}` | Название задачи |
| `{first_match}` | Первое совпадение |
| `{total_matches}` | Количество совпадений |
| `{matches_list}` | Список всех совпадений |
| `{group_1}`, `{group_2}` | Группы из RegExp |

## Примеры использования

### Мониторинг цены товара
1. URL: `https://shop.example.com/product/123`
2. RegExp: `Цена: (\d+) руб`
3. Шаблон: `Цена изменилась: {group_1} руб`
4. Условие: При изменении
5. Частота: 3600 сек (1 час)

### Проверка наличия товара
1. URL: `https://shop.example.com/product/123`
2. RegExp: `В наличии`
3. Шаблон: `Товар появился в наличии!`
4. Условие: При совпадении
5. Частота: 300 сек (5 минут)

## Ограничения

- Максимум 50 задач на пользователя
- Минимальный интервал проверки: 10 секунд
- Максимум 1000 записей истории на задачу
- Таймаут HTTP-запросов: 30 секунд (настраивается)
